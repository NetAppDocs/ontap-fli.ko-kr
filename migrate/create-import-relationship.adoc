= ONTAP FLI LUN 가져오기 관계를 생성합니다
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
외부 LUN이 검색되어 외부 LUN으로 표시되면 LUN 가져오기 관계를 생성하여 외부 LUN을 대상 ONTAP LUN과 연결할 수 있습니다. 대상 ONTAP LUN은 외부 디스크 식별자를 사용하여 생성되므로 소스와 일치하도록 크기가 자동으로 조정됩니다. 대상 LUN은 의도된 프런트 엔드 프로토콜 및 전환 계획에 따라 적절한 이니시에이터 그룹에 매핑된 다음 가져오기 설정 전에 안전 조치로 오프라인 상태로 전환됩니다.

다음을 수행하려면 설명된 대로 `lun create` 및 `lun import create` 명령을 사용하십시오.

* 외부 디스크 식별자를 사용하여 대상 ONTAP LUN을 생성하면 대상 크기가 소스와 일치하도록 자동으로 조정됩니다.
* 의도된 프런트 엔드 프로토콜을 기반으로 적절한 igroup을 정의하고 전환 계획에 따라 대상 LUN을 매핑하여 호스트 액세스를 준비합니다.
* 가져오기 설정을 시작하기 전에 안전 조치로 대상 LUN을 오프라인으로 전환하십시오.
* 외부 디스크와 대상 ONTAP LUN을 영구적으로 연결하고 후속 데이터 복사 작업을 위해 준비하는 LUN 가져오기 관계를 생성합니다.


.이 작업에 관하여
ONTAP 9.17.1 버전부터 ASA r2 시스템에서 FLI 오프라인 마이그레이션을 이용한 외부 LUN 데이터 마이그레이션이 지원됩니다. ASA r2 시스템은 스토리지 계층 구현에서 다른 ONTAP 시스템(ASA, AFF, FAS)과 다릅니다. ASA r2 시스템에서는 스토리지 유닛(LUN 또는 네임스페이스)을 생성할 때 볼륨이 자동으로 생성됩니다. 각 볼륨에는 하나의 스토리지 유닛만 포함됩니다. 따라서 ASA r2 시스템에서는 LUN 생성 시  `-path` 옵션에 볼륨 이름을 포함할 필요 없이 스토리지 유닛 경로를 포함해야 합니다.

.단계
. 외부 디스크와 일치하도록 크기가 조정된 대상 NetApp LUN을 생성합니다.
+
이렇게 하면 NetApp LUN과 외부 LUN의 크기가 동일해집니다.

+
[source, cli]
----
lun create -vserver <SVM> -path /vol/<VOL>/<lun_name> -ostype <ostype> -foreign-disk <foreign_serial>
----
. igroup을 생성하고 대상 LUN을 매핑합니다(프로토콜은 프런트엔드 계획에 따라 다름).
+
[source, cli]
----
igroup create -vserver <SVM> -igroup <igroup> -protocol <iscsi|fcp|mixed> -ostype <ostype> -initiator <host_initiator>
----
. NetApp 스토리지 어레이 LUN을 호스트에 매핑합니다.
+
[source, cli]
----
lun map -vserver <SVM> -path /vol/<VOL>/<LUN_NAME> -igroup <igroup>
----
. 가져오기 생성 전에 LUN을 오프라인 상태로 유지하십시오(모범 사례).
+
[source, cli]
----
lun offline -vserver <SVM> -path /vol/<VOL>/<LUN_NAME>
----
. 가져오기 관계를 생성합니다.
+
[source, cli]
----
lun import create -vserver <SVM> -foreign-disk <foreign_serial> -path /vol/<VOL>/<lun_name>
----
. 가져오기 관계가 존재하는지 확인합니다.
+
[source, cli]
----
lun import show -vserver <svm_name>
----
+
[NOTE]
====
이 단계에서 LUN을 온라인 상태로 전환하지 않으면 워크플로는 오프라인 FLI로 간주되며 NetApp 스토리지 어레이 LUN을 호스트가 인식하지 못하고 I/O 작업을 수행할 수 없게 됩니다.

** 오프라인 FLI는 가져오기 기간 동안 호스트가 오프라인 상태를 유지함을 의미합니다. ONTAP는 애플리케이션이 중지된 상태에서 외부 LUN에서 NetApp LUN으로 블록 복사를 수행하며, 호스트 복구 및 전환 작업은 가져오기 및 유효성 검사가 완료된 후에만 수행됩니다.
** 온라인 FLI는 호스트를 한 번만 전환하면 백그라운드에서 가져오기가 계속 진행되는 방식입니다. 일반적으로 전환 중에는 짧은 시간 동안만 서비스가 중단되며, 호스트 복구 및 전환 작업은 가져오기가 완료되기 전에 이루어지고, 그동안 ONTAP는 데이터 복사를 계속 진행합니다.


ONTAP 워크플로 관점에서 핵심 가져오기 명령은 두 모드 모두에서 동일합니다(`lun import create/start/show/verify/delete`. 실제 동작상의 차이점은 LUN 상태 및 타이밍에 있습니다. 오프라인 FLI에서는 대상 LUN이 일반적으로 완료 및 제시 준비가 될 때까지 오프라인 상태로 유지되는 반면, 온라인 FLI에서는 백그라운드 복사 중에 호스트 액세스를 활성화하기 위해 대상 LUN을 더 빨리 온라인 상태로 전환하여 제시할 수 있습니다.

====
. 온라인 FLI의 경우 LUN을 온라인 상태로 전환하십시오.
+
[source, cli]
----
lun online -vserver <SVM> -path /vol/<VOL>/<lun_name>
----
+
LUN이 온라인 상태가 되면 호스트에서 LUN을 검색하고 필요한 호스트별 수정 작업을 수행한 다음 LUN에서 워크로드를 시작할 수 있습니다.
