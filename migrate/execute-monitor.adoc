= FLI LUN 가져오기 실행 및 모니터링
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
가져오기 관계가 생성되면 lun import start를 사용하여 외부 LUN에서 대상 ONTAP LUN으로 백그라운드 데이터 가져오기를 시작하고, lun import show를 사용하여 진행 상황을 추적할 수 있습니다. 이 명령은 현재 작업(가져오기 또는 검증 등), 관리 및 운영 상태, 완료율을 보고하므로 작업이 완료될 때까지 모니터링할 수 있습니다.

호스트 I/O와 가져오기 작업이 동일한 백엔드 리소스를 놓고 경쟁하는 온라인 환경에서는 lun import pause 명령을 사용하여 체크포인트를 손실 없이 복사 스트림을 중지한 다음, 부하와 지연 시간이 안정화되면 lun import resume 명령을 사용하여 마지막 체크포인트부터 다시 시작할 수 있습니다. 이는 애플리케이션 응답 시간을 안정적으로 유지하면서 활성 복사 스트림을 조절하는 데 권장되는 제어 방법입니다.

.이 작업에 관하여
ONTAP 9.17.1 버전부터 ASA r2 시스템에서 FLI 오프라인 마이그레이션을 이용한 외부 LUN 데이터 마이그레이션이 지원됩니다. ASA r2 시스템은 스토리지 계층 구현에서 다른 ONTAP 시스템(ASA, AFF, FAS)과 다릅니다. ASA r2 시스템에서는 스토리지 유닛(LUN 또는 네임스페이스)을 생성할 때 볼륨이 자동으로 생성됩니다. 각 볼륨에는 하나의 스토리지 유닛만 포함됩니다. 따라서 ASA r2 시스템에서는 LUN 생성 시  `-path` 옵션에 볼륨 이름을 포함할 필요 없이 스토리지 유닛 경로를 포함해야 합니다.

.단계
. 권한 수준을 고급으로 설정합니다.
+
[source, cli]
----
set -privilege advanced
----
. LUN 가져오기를 시작합니다.
+
[source, cli]
----
lun import start -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
. 가져오기 진행 상황을 모니터링합니다.
+
[cols="2"]
|===
| 명령 | 설명 


| SVM에서 모든 가져오기 표시  a| 
[source, cli]
----
lun import show -vserver <SVM_NAME>
----


| 특정 LUN 가져오기를 표시합니다  a| 
[source, cli]
----
lun import show -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----


| 상세 보기(더 많은 필드 포함)  a| 
[source, cli]
----
lun import show -instance
----
|===
. 호스트 지연 시간에 영향을 미치는 경우(일반적인 온라인 제어), 일시 중지/재개:
+
[cols="2"]
|===
| 명령 | 설명 


| 가져오기 일시 중지(체크포인트 유지)  a| 
[source, cli]
----
lun import pause -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----


| 일시 중지되었는지 확인합니다  a| 
[source, cli]
----
lun import show -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----


| 마지막 체크포인트에서 가져오기 재개  a| 
[source, cli]
----
lun import resume -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----


| 다시 실행 중인지 확인하십시오  a| 
[source, cli]
----
lun import show -vserver <SVM_NAME> -path /vol/<VOL>/<LUN_NAME>
----
|===
