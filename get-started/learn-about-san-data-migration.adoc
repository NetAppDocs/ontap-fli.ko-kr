= ONTAP로 SAN 데이터 마이그레이션에 대해 알아보십시오
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SAN 데이터 마이그레이션은 종종 촉박한 다운타임 기간, 운영 위험 및 제한된 전문 리소스로 인해 제약을 받습니다. ONTAP Foreign LUN Import(FLI)는 외부(타사) 어레이 LUN에서 ONTAP 네이티브 LUN으로 데이터를 복사하는 내장 블록 마이그레이션 기능으로, 오프라인 및 온라인 마이그레이션 방식을 모두 지원합니다. ONTAP는 전문 서비스 주도 마이그레이션을 요구하지 않지만, 특히 복잡한 환경의 경우 범위 설정, 계획 수립 및 교육을 위해 NetApp 또는 자격을 갖춘 파트너를 참여시키는 것이 여전히 강력히 권장됩니다.



== 적절한 마이그레이션 접근 방식 선택(상위 수준)

실용적인 마이그레이션 전략은 하나의 주요 방법을 사용하고, 필요에 따라 다른 접근 방식을 보완적으로 활용할 수 있습니다. ONTAP로의 SAN/LUN 수준 마이그레이션의 경우 FLI가 일반적으로 강력한 선택이지만, 일부 시나리오에서는 설계상 무중단 방식인 대안이 더 적합할 수 있습니다.

.일반적인 옵션:
FLI(Foreign LUN Import):: 온라인 및 오프라인 워크플로우를 통한 ONTAP으로의 블록/LUN 수준 마이그레이션.
Data Transfer Appliance(DTA):: SAN에 연결된 어플라이언스 모델로, 마이그레이션된 TB당 라이센스가 부여되며 오프라인 및 온라인 마이그레이션을 모두 지원합니다.
호스트 OS 또는 애플리케이션 기반 마이그레이션:: 플랫폼에서 기본 이동성을 제공하거나 호스트 또는 애플리케이션 제어가 선호되는 경우에 유용합니다(예: VMware 스토리지 이동성 및 호스트 기반 복사 방식).




== 마이그레이션 기간 계획

초기 계획 단계에서만 일반적으로 인용되는 벤치마크 가정은 총 ~2TB 용량의 LUN 8개를 사용하는 호스트의 경우 호스트당 ~5시간이며, 이는 대략 ~400GB/시간에 해당합니다. 이는 의도적으로 대략적인 계획 수치이며 모든 환경에 정확히 들어맞지는 않을 수 있습니다. 정확한 예측을 위해서는 다양한 규모의 테스트 마이그레이션을 여러 번 실행하여 실제 스토리지, 네트워크 및 호스트 환경에서 처리량을 측정하는 것이 좋습니다.



== 권장 도구(데이터 수집, 호환성, 자동화, 검증)

서비스 도구는 환경 데이터를 수집하고, 지원 가능성을 검증하며, 계획 및 실행 과정에서 발생하는 인적 오류를 줄이는 표준화된 방법을 제공합니다.

Active IQ OneCollect:: 스토리지, 호스트, 패브릭 및 스위치에서 데이터 수집을 수행하여 문제 해결, 검증, 마이그레이션 계획 및 업그레이드 평가를 지원합니다.
상호 운용성 매트릭스 툴(IMT):: NetApp 및 타사 구성 요소 전반에 걸쳐 지원되는 구성을 확인할 수 있는 웹 기반 유틸리티입니다.
Solaris Relabeler(해당되는 경우):: 블록 수준 마이그레이션 후 시스템에 업데이트된 조회 정보가 반영되도록 마이그레이션 후 VTOC 디스크 레이블을 업데이트합니다.
SAN LUN Migrate(FLI 프로젝트용 sanlunmigrate):: 프로덕션 마이그레이션을 수행하기 전에 고객 환경에서 외부 어레이/LUN 동작의 유효성을 검사하기 위해 검증 방식을 사용하십시오. 이는 전송 또는 공급업체 동작이 다를 경우 특히 중요합니다.
