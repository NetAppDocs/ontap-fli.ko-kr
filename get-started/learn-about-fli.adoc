= ONTAP Foreign LUN Import에 대해 알아보십시오
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Foreign LUN Import(FLI)는 외부 어레이 LUN에서 ONTAP LUN으로 사용자 데이터를 복사하여 블록 레벨(LUN) 데이터 마이그레이션을 가능하게 하는 ONTAP 기능입니다. ONTAP는 SCSI 이니시에이터 역할을 하여 외부 LUN에서 데이터를 읽고 대상 ONTAP LUN에 기록함으로써 마이그레이션을 수행합니다.

FLI는 LUN 레벨에서만 작동하며 SAN 마이그레이션을 위해 설계되었습니다. 파일 레벨 마이그레이션 도구는 아닙니다(NFS/SMB와 같은 NAS/파일 프로토콜 마이그레이션에는 다른 접근 방식이 필요합니다). FLI는 오프라인 및 온라인 마이그레이션을 모두 지원합니다. 오프라인 모드에서는 가져오기 작업이 진행되는 동안 호스트 I/O가 중지됩니다. 온라인 모드에서는 가져오기 작업이 백그라운드에서 진행되는 동안 호스트 작업이 잠시 중단된 후 재개됩니다.

외부 LUN은 ONTAP 스토리지 스택에 나타나며, 실수로 덮어쓰거나 소유권이 이전되는 것을 방지하기 위해 명시적으로 외부 LUN으로 표시해야 합니다. FC FLI는 FC 백엔드 연결(이니시에이터 포트, 영역 설정/마스킹)을 사용하는 반면, iSCSI FLI는 백엔드에서 클러스터 간 연결에 바인딩된 iSCSI 소프트웨어 이니시에이터를 사용하고, 마이그레이션 후 호스트 연결(프런트엔드) 프로토콜은 설계에 따라 FC 또는 iSCSI를 사용할 수 있습니다.

.오프라인 및 온라인 FLI 비교
다음은 오프라인 마이그레이션 방식과 온라인 마이그레이션 방식의 주요 차이점을 요약한 것입니다.

[cols="3"]
|===
| 영역 | 오프라인 FLI | 온라인 FLI 


| 가져오기 중 호스트 I/O | 가져오기 작업이 진행되는 동안 호스트 I/O가 중지됩니다. | 호스트 I/O는 가져오기 복사가 진행되는 동안(전환 후) 실행될 수 있습니다. 


| 다운타임 프로필 | 가져오기 기간 동안 시스템 가동 중지 시간이 필요합니다. | 대부분 무중단으로 설계되었으며, 일반적으로 전환 시 짧은 운영 중단만 필요합니다. 


| 운영 사용 사례 | 더 긴 다운타임이 허용되거나 애플리케이션 또는 호스트 모델에서 필요한 경우에 가장 적합합니다. | 가동 중지 시간을 최소화해야 하고 백그라운드 복사 작업을 허용할 수 있는 경우에 가장 적합합니다. 
|===
.오프라인 FLI 워크플로우
오프라인 워크플로에서는 가져오기 전체 기간 동안 호스트 연결이 끊어집니다.

image::new-offline-fli.png[오프라인 FLI 워크플로우]

.온라인 FLI 워크플로우
온라인 워크플로우에서 호스트는 가져오기 작업이 복사 단계에서 완료 단계로 전환되는 짧은 전환 기간 동안에만 연결이 끊어집니다.

image::new-online-fli.png[오프라인 FLI 워크플로우]
