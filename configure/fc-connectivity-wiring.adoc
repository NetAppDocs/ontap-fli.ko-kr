= ONTAP FLI(Foreign LUN Import) 연결 및 배선
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
FC를 통한 외부 LUN 가져오기는 대상 ONTAP 컨트롤러가 검색 및 가져오기 중에 외부 어레이 LUN에 안정적으로 액세스할 수 있도록 이중 패브릭 SAN 설계를 기반으로 합니다. 지원되는 배포 환경에서는 사용 중인 각 스토리지 컨트롤러에 두 패브릭 모두에 기본 활성 경로가 있어야 합니다. 즉, 소스 어레이 타겟 포트와 가져오기를 수행하는 노드의 ONTAP 이니시에이터 포트가 패브릭 A와 패브릭 B의 공통 영역에 있어야 합니다. ONTAP 클러스터의 모든 노드를 포함할 필요는 없으며, 실제로 LUN을 가져오고 마이그레이션할 노드만 포함하면 됩니다. 간접 경로도 작동할 수 있지만, 소스와 대상 간에 활성 최적화 경로를 사용하는 것이 가장 좋습니다.



== ONTAP FLI 권장 FC 배선 및 조닝 사례

* 이중화를 위해 이중 패브릭(A 및 B)을 사용하십시오.
* 사용 중인 각 컨트롤러의 기본 활성 경로가 두 패브릭 모두에 존재하는지 확인하십시오.
* 가능한 경우 단일 장애 지점을 방지하기 위해 최소 2개의 ONTAP 이니시에이터 포트와 2개의 소스 어레이 타겟 포트를 사용하십시오.
* ONTAP 이니시에이터 포트를 호스트와 영역 설정하지 마십시오. ONTAP 이니시에이터 포트는 가져오기 및 검색을 위해 소스 어레이 타겟 포트에 연결하는 데만 사용됩니다.
* 가져오기에 참여하는 ONTAP 노드만 영역으로 지정하면 되고, 클러스터 전체를 영역으로 지정할 필요는 없습니다.
