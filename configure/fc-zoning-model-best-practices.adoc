= ONTAP Foreign LUN Import 권장 FC 배선 및 조닝 사례
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Foreign LUN Import(FLI)를 위해서는 소스 LUN이 ONTAP에서 액세스할 수 있어야 하며, 이는 소스 타겟 포트를 ONTAP 이니시에이터 포트에 영역화함으로써 달성됩니다. 기존의 소스-호스트 영역은 일반적으로 마이그레이션 중에 수정되지 않으며 마이그레이션 후 제거하거나 비활성화할 수 있습니다.

.표준 FC FLI 시나리오는 4개의 서로 다른 영역을 사용합니다.
* 영역 1: 소스 스토리지에서 대상 스토리지로(Fabric A)
* 영역 2: 소스 스토리지에서 대상 스토리지로(Fabric B)
* 영역 3: 호스트에서 대상 스토리지로(패브릭 A)
* 영역 4: 호스트에서 대상 스토리지로(패브릭 B)


.조닝 모범 사례
* 동일한 영역에서 소스 타겟 포트와 대상 타겟 포트를 혼합하여 사용하지 마십시오.
* 동일한 존에서 대상 이니시에이터 포트와 호스트 포트를 혼합하지 마십시오.
* 동일한 영역에서 대상 타겟 포트와 대상 이니시에이터 포트를 혼합하지 마십시오.
* 가능한 경우 이중화를 위해 컨트롤러당 최소 2개의 포트가 있는 영역을 구성하십시오.
* 단일 이니시에이터 및 단일 타겟 조닝을 선호합니다.
* FLI 워크플로에서 요구하는 영역 모델에 맞는 적절한 물리적 포트 WWPN을 사용하십시오. 마이그레이션 영역 지침에서 명시적으로 허용되지 않는 경우 NPIV WWPN을 사용하지 마십시오.

NOTE: 소스 타겟을 대상 이니시에이터에 맞게 영역화한 후에는 검색 명령을 통해 ONTAP에서 소스 스토리지가 표시되어야 합니다. 어레이가 처음 나타나지 않는 경우 일부 환경에서는 ONTAP 이니시에이터에 연결된 스위치 포트를 재설정해야 할 수 있습니다.
