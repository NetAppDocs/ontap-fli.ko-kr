= ONTAP FLI HP 호스트 해결
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
HP 호스트에서 ONTAP FLI 마이그레이션을 실행하기 전에 다음 사전 마이그레이션 단계를 완료하십시오. 오프라인 마이그레이션과 온라인 마이그레이션에 대한 단계가 각각 다릅니다.



== 사전 마이그레이션 - 오프라인 FLI

이러한 단계는 온라인 및 오프라인 FLI 모두에 공통적으로 적용됩니다. 호스트에서 소스 LUN의 프로비저닝을 해제하기 전에 다음 단계를 완료했는지 확인하십시오.

[cols="3"]
|===
| 파일 시스템(LVM 및 Non-LVM) | 호스트 클러스터(SCSI3 PR) | SANBOOT 


| 1단계: 엔드투엔드 블록 장치 구성 수집: 다중 경로 상태/경로, 부팅 LUN의 LVM 레이아웃, 그리고 올바른 부팅 장치 및 부팅 매개변수를 확인하는 UEFI 설정. | 1단계: 전체 클러스터 세부 정보와 마이그레이션에 관련된 디스크를 캡처합니다. | 1단계: 엔드투엔드 SAN 부팅 구성 수집: 다중 경로 상태/경로, 부팅 LUN의 LVM 레이아웃, 그리고 올바른 부팅 장치 및 부팅 매개변수를 확인하는 UEFI 설정. 


| 2단계: 각 데이터 디바이스(LVM/Non-LVM)에 대한 /etc/fstab 항목이 올바른지 확인합니다. | 2단계: 권장 절차에 따라 클러스터 서비스를 정상적으로 종료합니다. | 2단계: 데이터 장치(lvm/nonlvm)를 포함한 SAN 부팅 장치에 대한 /etc/fstab 항목이 올바른지 확인합니다 


| 3단계: 대상 마운트 지점에 대한 모든 작업이 일시 중지되었는지 확인합니다. 블록 장치에 접근하는 프로세스가 남아 있지 않은지 확인합니다. 필요한 경우 열려 있는 프로세스 핸들을 비우거나 종료합니다. | 3단계: 다중 경로 장치와 연결된 SCSI 블록 장치를 제거합니다. | 3단계: UEFI 셸에서 장치 매핑 테이블을 확인하고 스토리지 장치에서 hpux로 부팅합니다 


| 4단계: 파일 시스템을 마운트 해제하고 LVM 볼륨 그룹 구성 백업을 수행합니다. VG를 비활성화하고 VG를 내보냅니다 | 4단계: 호스트를 다시 스캔하여 기존 디바이스 매핑에 오래된 항목이 없는지 확인합니다. | 4단계: 기본 부팅 장치가 스토리지 장치로 설정되어 있는지 확인하여 이후 재부팅 시 자동으로 부팅되도록 합니다. 


| 5단계: 오래된 SCSI 디바이스 파일을 모두 제거합니다. |  | 5단계: 호스트를 재부팅하여 위의 변경 사항이 정상적으로 적용되는지 확인합니다. 호스트가 정상적으로 부팅되면 FLI 마이그레이션을 위한 호스트 사전 복구 작업이 완료된 것입니다. 


| 6단계: 호스트에 업데이트된 장치 목록이 표시되도록 sci 버스를 다시 스캔합니다(예: ioscan -fnC disk). |  | 6단계: 이제 ONTAP FLI 워크플로우를 진행하기 위해 sanboot 호스트를 종료합니다. 
|===


== 사전 마이그레이션 - 온라인 FLI

아래는 온라인 FLI의 구체적인 단계이지만, 이러한 단계는 사전 마이그레이션 오프라인 FLI의 단계를 먼저 수행해야 합니다. 다음 단계는 NetApp LUN이 호스트에 프로비저닝된 후에 수행해야 합니다.

[cols="4"]
|===
| 파일 시스템 | 블록 디바이스 | LVM(FS) | Sanboot 


| 1단계: 매핑된 모든 HP-UX 호스트에서 스토리지를 다시 검색합니다 | 1단계: 매핑된 모든 HP-UX 호스트에서 스토리지를 다시 스캔합니다. | 1단계: 매핑된 모든 HP-UX 호스트에서 스토리지를 다시 검색합니다 | 1단계: 서버 전원 켜기 


| 2단계: 다중 경로 디바이스 식별 | 2단계: 다중 경로 디바이스 식별 | 2단계: 다중 경로 디바이스 식별 | 2단계: UEFI 셸에서 장치 매핑 테이블을 확인하고 새 스토리지 장치에서 hpux로 부팅합니다 


| 3단계: 파일 시스템 세부 정보 확인 | 3단계: 다중 경로 디바이스 ID를 식별하고 해당 애플리케이션에 제공합니다. | 3단계: 새 LVM 장치 파일이 올바르게 생성되었는지 확인합니다 | 3단계: 기본 부팅 장치를 새 스토리지 LUN으로 수정 LUN 가져오기의 일부인 새로 매핑된 LUN에서 부팅하도록 호스트를 다시 시작합니다 


| 4단계: 장치를 각각의 폴더에 다시 마운트합니다 |  | 4단계: VG/LVM 가져오기 |  


|  |  | 5단계: VG 활성화 |  


|  |  | 6단계: VG에 존재하는 LV 목록을 작성하십시오 |  


|  |  | 7단계: LVM을 각각의 폴더에 다시 마운트합니다 |  
|===

NOTE: 최신 및 업데이트된 명령 목록은 OS 공급업체 설명서를 참조하십시오.

.단계
. 스토리지 디바이스를 식별합니다.
+
[Source]
----
ioscan -fnC disk
sanlun lun show all (Works on Netapp Storage)
sanlun lun show all -pv
sanlun lun show -wwpn
----
. 볼륨 그룹과 파일 시스템을 식별합니다.
+
[Source]
----
vgdisplay -v /dev/vg_name
vgdisplay /dev/vg_name
lvdisplay /dev/vg_name/lv_name
----
. 파일 시스템을 마운트 해제하고 VG를 내보냅니다.
+
[Source]
----
bdf
umount /mount_point
vgchange -a n /dev/vg_name
vgexport /dev/hpux_vg_online
----
. 분리 후 다시 스캔하여 깨끗한 상태인지 확인하십시오.
+
[Source]
----
insf -e
ioscan -fnC disk
----
