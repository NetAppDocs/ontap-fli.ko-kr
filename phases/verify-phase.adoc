= ONTAP 외부 LUN 가져오기(FLI) 검증 단계
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
검증 단계에서는 마이그레이션이 계획대로 완료되었는지 확인하고 마이그레이션 후 정리 작업에 집중합니다. 여기에는 애플리케이션 동작 검증, ONTAP LUN에 대한 호스트 액세스 및 경로 확인, 감사 및 문제 해결을 위한 마이그레이션 로그 수집, 마이그레이션에만 사용된 임시 연결 및 구성 제거 등이 포함됩니다.

정리 작업의 일환으로 더 이상 필요하지 않은 소스 측 이니시에이터 레코드와 영역 설정을 제거하고, 운영 팀이 명확한 "완성된" 상태를 파악할 수 있도록 최종 구성을 문서화합니다.

iSCSI FLI의 경우, 검증 단계에서는 인터클러스터 LIF에 생성된 소프트웨어 iSCSI 이니시에이터 세션과 이니시에이터를 제거하는 작업도 포함되어야 합니다. 인터클러스터 LIF가 외부 LUN 가져오기만을 위해 생성된 경우, 해당 LIF도 삭제해야 합니다.



== ONTAP FLI 검증 단계 작업

다음은 각 구성 요소에 대한 ONTAP FLI 검증 작업입니다.

[cols="1,2"]
|===
| 구성 요소 | 작업 


| 호스트  a| 
* 마이그레이션된 LUN에 대해 애플리케이션 수준의 유효성 검사를 수행합니다. (파일 시스템 일관성 검사, 체크섬 오류, dbcc checkdb(MS SQL), rman(Oracle))
* 마이그레이션 및 복구 작업 후 예상되는 모든 파일 시스템, 데이터 저장소, 클러스터 디스크에 액세스할 수 있고 안정적인지 확인하십시오.
* IO 경로 개수와 경로 유형이 예상대로 표시되는지 확인하십시오.




| FC FLI 패브릭  a| 
* 초기 컷오버 중에 아직 제거되지 않은 경우 활성 존 세트 또는 이에 상응하는 액세스 규칙에서 호스트-소스 스토리지 존을 제거합니다.
* 마이그레이션을 위해 특별히 생성된 소스-대상 연결 영역을 제거합니다.
* 롤백 및 감사 목적으로 업데이트된 마이그레이션 후 영역 세트를 백업합니다.




| iSCSI FLI 네트워크  a| 
* 마이그레이션을 위해 생성된 iSCSI FLI 백엔드 연결 객체를 제거합니다.
* iSCSI 이니시에이터 세션을 삭제해야 하며, 클러스터 간 LIF에 생성된 iSCSI 이니시에이터 객체도 삭제해야 합니다.




| NetApp 스토리지 어레이(대상)  a| 
* 마이그레이션 후 스토리지의 정확성을 검증하십시오. 필요한 경우 호스트 또는 사용 사례에 따라 LUN 정렬을 확인하십시오.
* 감사 및 문제 해결을 위해 FLI 마이그레이션 가져오기 로그를 수집합니다. 마이그레이션이 완료되면 AutoSupport를 실행합니다.
* 마이그레이션 기간 동안 롤백 용도로만 사용되는 수동으로 생성된 Snapshot 복사본을 모두 제거하십시오.
* 소스 어레이 객체 및 외부 디스크 연결 세부 정보와 같이 더 이상 필요하지 않은 소스 어레이 구성 아티팩트를 ONTAP에서 제거합니다. storage disk show 명령을 사용하여 동일한 내용을 확인합니다.
* LUN 경로, 매핑, igroup, 이니시에이터 및 호스트 액세스 모델을 포함한 최종 ONTAP 구성을 문서화하십시오.




| 외부 스토리지 어레이(소스)  a| 
* 마이그레이션 중에 소스 LUN을 ONTAP에 제공하기 위해 생성된 대상 스토리지(NetApp) 이니시에이터 그룹 및 호스트 그룹을 제거합니다.
* 소스 액세스를 해제할 때 원래 호스트 매핑과 관련된 호스트 이니시에이터 그룹 및 호스트 레코드를 제거하십시오.
* 마이그레이션된 소스 LUN이 호스트에 이중으로 노출될 수 있는 방식으로 더 이상 제공되지 않도록 해야 합니다. 이는 매우 중요하며 마이그레이션 과정 전반에 걸쳐 반드시 확인해야 합니다.


|===
