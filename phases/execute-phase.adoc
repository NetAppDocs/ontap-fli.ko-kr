= ONTAP Foreign LUN Import(FLI) 실행 단계
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
실행 단계는 마이그레이션이 수행되는 단계입니다. 이 단계에서는 오프라인 또는 온라인 Foreign LUN Import를 실행하고 호스트 액세스를 외부 어레이에서 ONTAP 어레이로 전환하는 데 필요한 모든 작업이 포함됩니다.



== ONTAP FLI 실행 단계 작업

다음은 각 구성 요소에 대한 ONTAP FLI 실행 작업입니다.

[cols="1,2"]
|===
| 구성 요소 | 작업 


| 호스트  a| 
* 주요 재구성 작업을 수행하기 전에 호스트 이벤트 및 시스템 로그를 검토하여 문제를 조기에 파악하고 위험을 줄이십시오.
* 조기 전환 또는 문제 해결 전에 안정성을 확보하기 위해 필요에 따라 호스트를 재부팅하십시오.
* 전환 후 호스트 액세스를 검증합니다. 예상되는 모든 장치가 연결되어 있고, 다중 경로가 정상이며, 애플리케이션과 파일 시스템이 문제없이 시작되는지 확인합니다.




| iSCSI FLI 및 네트워크  a| 
* IC LIF가 외부 어레이 타겟 포털에 도달 가능한지 확인하고 가져오기 과정 전반에 걸쳐 백엔드 연결이 안정적으로 유지되는지 확인하십시오.
* iSCSI 세션이 정상 상태로 유지되는지 확인합니다. 이는 외부 LUN 검색 및 경로 가용성을 구동하기 때문입니다.




| 패브릭  a| 
* ONTAP 이니시에이터 포트가 검색 및 가져오기를 위해 외부 어레이 LUN에 액세스할 수 있도록 필요한 조닝 및 마스킹이 설정되어 있는지 확인하십시오.
* 전환 후 필요에 따라 영역 설정을 업데이트하여 호스트가 대상 ONTAP LUN에 액세스할 수 있도록 하고 나중에 정리 작업을 준비하십시오.




| NetApp 스토리지 어레이(대상)  a| 
* 가져오기 작업을 생성하기 전에 ONTAP에서 외부 LUN이 검색되었는지 확인하십시오.
* 검색이 확인되면 가져오기 관계를 생성하고 가져오기 작업을 시작합니다.
* 가져오기 진행 상황을 모니터링하고 예외를 처리하세요. 특히 호스트 지연 시간이 증가하는 경우, 필요에 따라 일시 중지 및 재개 기능을 사용하여 복사 스트림을 조절하세요.
* 전환 처리:
+
** 오프라인 FLI 전환은 가져오기가 완료된 후에 발생합니다.
** 온라인 FLI 호스트 전환은 가져오기가 백그라운드에서 계속 진행되는 동안 더 빨리 이루어집니다.






| 외부 스토리지 어레이(소스)  a| 
* 마이그레이션할 소스 LUN이 ONTAP 검색 및 가져오기를 위해 올바르게 표시되는지 확인하십시오(전송 방식에 따라 FC 마스킹 또는 iSCSI 타겟 가시성).
* 가져오기 과정 중에 호스트를 통해 소스 LUN에 액세스하지 않도록 하십시오. 이는 여러 소스에서 쓰기 작업이 발생하는 것을 방지하는 데 매우 중요합니다.
* 온라인 전환 중에 이중 표시를 방지하고 호스트가 ONTAP로 원활하게 전환되도록 필요에 따라 매핑 해제 및 재매핑 작업을 조정합니다.


|===
FC 및 iSCSI 백엔드 FLI 모두에서 실행 성공은 FC/iSCSI 세션 상태 유지 및 외부 LUN에 대한 전체 경로 이중화를 보장하여 안정적인 백엔드 연결을 유지하는 데 크게 좌우됩니다. 호스트 I/O(온라인)와 가져오기 복사본이 동일한 백엔드 리소스를 놓고 경쟁할 수 있으므로 호스트 지연 시간을 보호하기 위해 필요한 경우 가져오기 속도를 운영상 제한(예: 일시 중지 및 재개 사용)할 수 있으며, 마이그레이션은 일반적으로 사용량이 적은 시간대에 예약됩니다.
