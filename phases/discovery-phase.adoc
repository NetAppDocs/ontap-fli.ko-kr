= ONTAP Foreign LUN Import(FLI) 검색 단계
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP FLI 검색 단계는 마이그레이션 프로세스의 첫 번째 단계입니다. 원활한 마이그레이션을 위해서는 소스 환경, 연결 상태 및 대상 ONTAP 시스템에 대한 모든 필수 정보를 수집하는 것이 매우 중요합니다. 검색 단계는 마이그레이션 계획, 준비, 실행 및 검증을 위한 기반을 마련합니다.



== ONTAP FLI를 사용하여 iSCSI 마이그레이션을 준비하세요

ONTAP FLI 검색 단계에서는 마이그레이션을 완료하는 데 필요한 기술 세부 정보를 수집합니다.

* 마이그레이션 워크플로우 계획(온라인 vs 오프라인)
* 호스트 복구 작업을 안전하게 실행합니다
* 가져오기 관계를 생성하기 전에 ONTAP에서 외부 LUN의 연결 및 검색 여부를 확인하십시오


호스트, NetApp 컨트롤러 및 패브릭 전반에 걸쳐 구조화되고 반복 가능한 데이터 수집을 위해 NetApp Active IQ OneCollect를 사용할 수 있습니다. OneCollect는 마이그레이션 계획 및 검증을 위해 스토리지, 호스트, 패브릭 및 스위치에서 정보를 수집하도록 설계되었습니다.

FC FLI의 경우 검색 단계에서 외부 어레이 타겟 포트 WWPN을 캡처해야 합니다.

iSCSI FLI의 경우 검색 단계는 다음을 수행해야 합니다.

* 외부 어레이 타겟 포털 IP 및 타겟 IQN을 캡처합니다
* IC LIF 도달 가능성 확인
* 모든 사용 가능한 경로를 통해 외부 LUN이 검색되도록 풀 메시 iSCSI 세션(각 HA 노드가 각 대상 포털에 연결됨)을 계획하십시오




== ONTAP FLI 검색 단계 작업

각 구성 요소에는 데이터를 수집하거나 연결을 검증하는 일련의 작업이 있습니다.

[cols="1,2"]
|===
| 구성 요소 | 작업 


| 호스트  a| 
* 호스트 OS 버전, 다중 경로 스택 및 구성, 그리고 파일 시스템, 원시 장치 또는 클러스터 디스크와 같은 특수 사용 모델을 파악하여 호스트 문제 해결 및 전환 단계를 진행합니다.
* 계획 및 롤백을 위해 크기, 식별자 또는 일련 번호, 파티션 레이아웃 또는 오프셋, 각 LUN이 애플리케이션에서 사용되는 방식을 포함한 LUN 인벤토리를 캡처합니다.
* FC 프런트엔드 환경의 경우 호스트 이니시에이터 WWPN을 기록하십시오. 이는 전체 마이그레이션 및 전환 설계에 따라 영역 설정 및 마스킹에 사용됩니다.
* iSCSI 프런트엔드 환경을 마이그레이션 후에 사용하는 경우, 전환 후 호스트가 로그인에 사용할 호스트 이니시에이터 IQN과 타겟 포털 IP를 기록해 두십시오.




| 패브릭 및 네트워킹  a| 
* FC 환경의 경우:
+
** 호스트 및 소스 어레이 영역 간의 현재 조닝을 문서화하여 컷오버 후 안전하게 제거하거나 업데이트할 수 있도록 합니다.
** 스위치 식별자와 관련된 포트를 기록해 두십시오.


* iSCSI 환경의 경우:
+
** ONTAP IC LIF와 외부 어레이 대상 포털 간의 VLAN, 서브넷 및 라우팅 경로를 문서화합니다
** 방화벽이나 ACL로 인해 통신이 차단되지 않는지 확인하십시오.






| NetApp 스토리지 어레이(대상)  a| 
* iSCSI FLI를 위한 ONTAP 버전 및 ECV 준비 상태를 확인하십시오.
+

NOTE: 클러스터가 지원되는 수준이 아니면 이니시에이터 생성이 차단됩니다.

* LUN 소유 노드 및 해당 HA 파트너와 같이 참여할 노드를 식별하고 해당 노드에서 사용 가능한 인터클러스터 LIF를 기록합니다. 이중화를 계획하고, 두 HA 노드 모두에서 IC LIF, 이니시에이터 및 세션을 계획합니다.
* IC LIF IPspace와 서브넷을 기록하고 외부 타겟 포털에 도달할 수 있는지 확인합니다. 기존 IC LIF가 클러스터의 다른 곳에서 이미 사용 중인 경우 다른 서브넷에 새 IC LIF를 생성하지 마십시오.
* 계획 중에 적용해야 하는 대상 측 제약 조건(예: 가져오기 최대 크기, 512B 블록 크기 요구 사항, iSCSI FLI와 관련된 세션 및 가져오기 확장 제한)을 검증하십시오.




| 외부 스토리지 어레이(소스)  a| 
* 외부 어레이 타겟 포털 IP와 타겟 IQN을 식별하십시오. iSCSI FLI에서는 iSCSI 검색 세션이 지원되지 않으므로 이러한 정보는 사전에 명시적으로 알고 있어야 합니다.
* 소스 호스트에 매핑된 특정 LUN을 식별하고, LUN 크기 및 블록 크기를 확인하십시오. LUN이 안정적이고 확장되지 않았으며 가져오기 계획에 적합한지 확인하십시오.
* 호스트 액세스 및 마스킹 객체(예: 호스트 및 이니시에이터 레코드)를 기록하여 전환 중에 안전하게 매핑을 해제하거나 다시 매핑하고 롤백 준비 상태를 검증할 수 있습니다.
* 외부 어레이 검증을 계획하고 실행합니다.
+

NOTE: iSCSI FLI에는 현장 검증이 필요합니다. FC 검증을 거친 어레이가 iSCSI에 자동으로 검증된다고 가정하지 마십시오.



|===


== 권장 ONTAP FLI 검색 결과물

* 단일 인벤토리 시트 매핑: 호스트 → 소스 LUN → 외부 타겟 포털/IQN → ONTAP IC LIF/노드 → 대상 LUN으로 원활한 실행 및 롤백이 가능합니다.
* 문제가 발생할 경우 문제 해결 시간을 줄이기 위해 변경 실행 전에 캡처한 OneCollect 패키지(또는 이와 동등한 패키지).
